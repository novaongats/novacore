# NOVACore 教訓集

## 2026-03-02: NovaSync全面修正 + 全機能検証

### 発見された根本原因パターン

1. **forcePushAll の自動実行はデータ消失の主犯** — ログイン時や起動時にlocalStorageをFirebaseに無条件上書きすると、他デバイスの最新データが消える。自動pushは廃止し、個別push（変更時のみ）に限定すべき。

2. **バージョンマイグレーションでのデータ消去は危険** — 新デバイスでlocalStorageが空の状態でデータをremoveItemすると、その後のpushで空データがFirebaseに上書きされる。

3. **Firebase リスナーの重複登録** — `.off()` を呼ばずに再接続のたびに `.on()` を登録すると、リスナーが蓄積し同じイベントが複数回発火する。フラグで制御。

4. **配列データの丸ごと上書き** — 2台が同時に異なるレコードを追加すると片方が消える。idベースのマージが必要。

### 発見されたバグパターン

5. **関数エイリアスの不整合** — `escH` vs `escHtml`。新機能追加時に既存ユーティリティ関数名を確認せずに短縮名を使用。

6. **同名関数の二重定義** — `renderMonthlyReport` が月報用と発注用の2箇所で定義され、後者が前者を上書き。同じスコープ内の関数名は一意にすること。

7. **要素IDの不一致** — `monthlyTarget` が存在しないのに参照。コピペ時にIDを修正し忘れ。

8. **iframe間の関数名齟齬** — `handleCSVExport`/`handleExport`/`importInput` が別iframeの関数名・要素IDのまま残存。各iframeはスコープが独立しているため、コピペ時に関数名・要素IDを必ず確認。

9. **CSSセレクタの不一致** — JS側 `.sync-badge` vs HTML側 `.bdg-sync`。名前変更時に全参照箇所を検索して更新すること。

10. **@media print CSSの欠如** — 印刷ボタンがあるiframeに印刷用CSSが無いと、UI要素がそのまま印刷される。

11. **CSS規則の切り詰め** — 長い行を手動編集した際にCSSルールが途中で切れることがある。

### 印刷レイアウトパターン

12. **`!important` が print CSS を上書き** — `nc-embed-override` のような画面用スタイルシートで `!important` を使うと、`@media print` 内の通常ルールが負ける。画面専用ルールは `@media screen{}` で囲むこと。

13. **ポップアップウィンドウにCSS変数が引き継がれない** — `window.open()` で作った新ウィンドウには親のCSS変数 (`var(--text)` 等) が存在しない。インラインHTMLではハードコード値 (`#333` 等) を使うこと。

14. **ネストされた `@page` はCSS仕様上無効** — `#element { @page { size: A4; } }` はブラウザに無視される。条件付きページサイズ変更はJSでの動的制御が必要。

15. **`@page` ルール無し = ブラウザデフォルト** — 請求書のようなフォーマルな文書には必ず `@page { size: A4; margin: ... }` を定義すること。

16. **`print-color-adjust: exact` の明示** — 背景色や文字色が印刷で消える場合は `print-color-adjust: exact; -webkit-print-color-adjust: exact;` を追加。

17. **印刷時に非表示にすべき要素** — フィルタバー、追加ボタン、アクションボタンは必ず `.no-print` を付与。モーダルは `.modal`/`.modal-overlay` を print CSS で `display:none` にすること。

### CSS変数パターン

18. **`--nv-*` と `--nc-*` の混同** — `nv-*-fix` スタイルブロックが `--nv-*` 変数を参照していたが、定義は `--nc-*` のみ。CSS変数のプレフィックスは統一し、未定義変数は `var()` に渡さないこと。未定義の `var()` は無効値となり、プロパティが無視される。

19. **CSSの宣言順でのカスケード上書き** — `nv-*-fix` ブロックが `nc-modern` / `nova-unified` の後に来るため、同じ `!important` でも後者が勝つ。後から来るブロックの変数が未定義だと、正しい前のルールが上書きされて壊れる。

20. **`.btn-danger` 等のベーススタイル欠如** — `:hover` のみ定義して基本スタイルが無いと、背景色も文字色も未設定でボタンが見えない。hover だけでなく基本状態も必ず定義すること。

21. **AnCore の `:root` に `--bg-secondary` / `--red-light` 未定義** — CSS変数を使用する前に `:root` で必ず定義すること。変数追加時は `:root` ブロックとの整合性を確認。
